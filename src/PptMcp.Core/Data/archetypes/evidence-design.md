# Evidence Design (Supporting Claims)

When a slide makes a quantitative claim (ROI, growth, cost savings), don't just state the number — show HOW:

**Weak (text assertion):**
```
Big number: 300% ROI
Text below: "Average return across deployments"
```

**Strong (visual evidence):**
```
Big number: 300% ROI (hero, centered)
Below: waterfall or bridge showing: cost invested → savings year 1 → savings year 2 → total return
OR: comparison bar: "Before: $4.2M cost" vs "After: $1.1M cost" 
OR: 3 mini case cards with specific client metrics
```

For every major claim, include at least ONE of:
- Benchmark comparison (vs. industry, vs. competitors, vs. prior period)
- Driver breakdown (what contributed to the number)
- Trend line (how it changed over time)
- Third-party validation (source citation from analyst/auditor)

### ROI / Financial Claims (CRITICAL — Most Scrutinized)

ROI slides get the most executive pushback. A before/after panel is better than text, but a CFO will ask "show me the math." 

**MANDATORY ROI PROOF PATTERN (When prompt says "proving the math"):**

When the user specifically requests mathematical proof of ROI, **DO NOT use bullet-style prose**. Instead, use a structured calculation layout that makes the arithmetic auditable:

**Required sequence: Driver breakdown → Total value → Investment → ROI formula**

```
Driver breakdown (each with calculation):
  Platform cost savings: $4.2M → $1.1M = $3.1M annual savings
  Revenue acceleration: 14-day → real-time decisions = $1.8M captured revenue  
  License consolidation: 12 tools → 1 platform = $0.4M annual savings

Total value calculation:
  $3.1M + $1.8M + $0.4M = $5.3M total annual value

Investment:
  Initial implementation: $1.8M

ROI formula (show the math):
  ROI = (Total Value - Investment) / Investment × 100%
  ROI = ($5.3M - $1.8M) / $1.8M × 100% = 194% 
  Add Year 2+ value: +100% = 294% total ROI (3 years)
```

**Preferred visual formats for ROI proof:**
- **Bridge/Waterfall**: Start with investment cost, add each value driver as a bar, end with total return
- **Mini-waterfall**: Table format with calculation steps as rows, running total column
- **Equation block**: Mathematical formula with each variable defined and calculated

**CRITICAL: Arithmetic Consistency Check**
Before completing any ROI slide, verify that ALL numbers reconcile:
- Hero lockup number ("300% ROI") must match the calculated formula result
- Component values in body text must add up to the total value used in the formula  
- Footer formula (if shown) must use the exact same inputs and produce the hero number
- **TEST**: Can an executive reconstruct your hero number from the component breakdown? If not, fix the calculation.

**NEVER use for mathematical ROI proof:**
- Plain bullet points with text descriptions
- Vague assertions without specific numbers  
- "Multiple benefits include..." prose paragraphs
- Before/after comparisons without calculation steps
- **Inconsistent numbers**: Hero says "300%" but calculation shows "294%" without explanation

**Level 1 (minimum): Before/After comparison**
```
Left: Before metrics (cost, time, tools)
Right: After metrics (lower cost, faster, unified)
```

**Level 2 (better): Value driver build-up**
```
Hero number: 294% ROI
Below: 3 rows showing each value driver:
  Driver 1: Reduced platform cost: $4.2M → $1.1M = $3.1M saved
  Driver 2: Faster decisions: 14 days → real-time = $1.8M revenue captured
  Driver 3: Tool consolidation: 12 → 1 = $0.4M license savings
  TOTAL: $5.3M value on $1.8M investment = 294% ROI
```

**Level 3 (board-grade): Investment waterfall bridge**
```
Start: $1.8M investment
  + Platform savings: $3.1M
  + Revenue uplift: $1.8M
  + License consolidation: $0.4M
  = Total value: $5.3M
  → ROI: 294% (rounded to 300%)
```

**Source citation for financial claims must include:**
- Study/report name (not just "industry study")
- Year and sample size
- Whether figures are annualized, average, or case-specific
- Example: "Source: Industry ROI Study, 2024 (n=47 enterprise deployments, annualized 3-year returns)"
